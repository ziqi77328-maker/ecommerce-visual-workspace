# Ecommerce Visual Workspace Memory

This repository is the default long-term context for ecommerce visual main-image work, especially JD appliance main images, promotion posters, selling-point expression, product structure layouts, price/benefit modules, and AI image-generation prompts.

Before starting a related task, read these files in order:

1. `PROJECT_MEMORY.md`
2. `README.md`
3. `000-工作流/主图生成工作流.md`
4. `000-工作流/主图信息填写表.md`
5. `000-工作流/产品关键词匹配与优化规则.md`
6. `008-agent决策中心/01-主图决策树.md`
7. `008-agent决策中心/02-优先级评分系统.md`
8. `008-agent决策中心/05-强制锁定清单.md`
9. Relevant files under `001-项目案例库`, `003-产品结构排版库`, `004-卖点表达库`, `005-价格与权益表达库`, and `006-AI关键词库`

For image analysis or learning from external examples, use `007-案例学习与卖点提取系统/08-图片分析标准Prompt.md` and `007-案例学习与卖点提取系统/09-图片分析输出模板.md`.

When new reusable knowledge appears, place it in the narrowest matching library instead of duplicating it across multiple folders. Use `000-工作流/仓库内容归类与更新规则.md` before adding or moving knowledge.

Do not treat ecommerce main-image tasks as blank-slate design work. Reuse this repository's accumulated patterns, examples, decision rules, and prompt libraries first.

For every image generation or redesign, the forced lock checklist must be output and confirmed before any prompt or image generation. This applies to first drafts and every later revision.
