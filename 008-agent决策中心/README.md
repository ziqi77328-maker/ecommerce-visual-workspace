# 008-agent决策中心

这个目录负责把现有知识库变成可执行的选择系统。

现有仓库已经有工作流、案例、场景、版式、卖点、价格权益和 AI 提示词，但它们更像资料库。agent 决策中心的作用是：在生成前先判断该用哪些资料、哪些信息优先、哪些风险要拦截；在生成后记录失败原因，避免反复踩同一个坑。

---

## 是否需要建立

需要。

原因：

- 现有知识库已经足够丰富，但缺少统一入口来决定“先用哪个、后用哪个”。
- 同一产品可能同时命中多个卖点、场景和版式，没有优先级系统时容易堆信息。
- AI 生成图容易出现产品变形、文字塑料感、背景过度特效、证据素材被重绘等问题，需要失败数据库持续校正。
- 电商主图不是单纯好看，而是要在真实产品、卖点证据、价格权益和平台缩略图之间做取舍，需要决策树。

---

## 使用顺序

每次做主图时，先走这里，再进入其它库。

1. 用户给出产品、卖点、活动或参考图后，先用 [01-主图决策树.md](01-主图决策树.md) 对照检查缺少哪些内容。
2. 如果用户只给了一部分卖点、价格或画面要求，必须先问用户“是否就这些，是否还需要补充”，不能默认内容完整。
3. 发现缺少关键信息时，先和用户核对，不直接编造参数、价格、权益和材质。
4. 用户确认内容完整后，生成图片前必须先列出图片中将使用的全部文字内容，并检查是否有重复卖点。
5. 画面文字清单必须给用户审核，用户确认后才允许生成图片。
6. 用 [02-优先级评分系统.md](02-优先级评分系统.md) 给卖点、素材、版式、价格权益和风险打分。
7. 用 [04-AI感降低策略.md](04-AI感降低策略.md) 修正提示词和出图标准。
8. 出图失败或效果不自然时，写入 [03-失败数据库.md](03-失败数据库.md)。
9. 再回到 `000-工作流/主图生成工作流.md` 执行生成。

---

## 决策中心输出

每次生成前，agent 应输出下面 8 项：

```text
1. 主图目标：新品/促销/功能教育/高端质感/场景种草
2. 推荐版式：左文右图/居中主视觉/强促销浅背景左文右品
3. 核心卖点：只保留 1 个
4. 辅助卖点：保留 3-5 个，不能和标题、证据区、价格区重复
5. 固定素材：必须调用的常用图片、官方图、证据贴片
6. 风险拦截：AI感、产品变形、参数臆造、证据错误、信息拥挤
7. 缺失信息：需要向用户核对的内容
8. 画面文字清单：图中所有文字、位置、用途、重复卖点检查结果
```

## agent 默认原则

- 先求真实，再求好看。
- 先保证产品不变形，再加光影和背景。
- 先证明核心卖点，再补充装饰氛围。
- 先保留成交信息，再压缩弱卖点。
- 能用官方图、常用图片和真实素材时，不重新生成关键证据。
- 用户只给部分卖点时，必须先确认“是否就这些，是否还需要补充”。
- 生成图片前，必须先列出图中全部文字内容，并检查重复卖点。
- 画面文字预览必须经过用户确认后，才允许生成图片。
- 用户指出图片需要调整时，必须把失败图、最终图和调整原因写入失败数据库。
- 任何失败都要归因到失败数据库，而不是只重新试一张。
