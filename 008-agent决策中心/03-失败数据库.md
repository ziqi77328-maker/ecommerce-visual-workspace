# 失败数据库

用于记录每次出图失败的原因、修正方式和下次拦截规则。

失败数据库的目的不是存废图，而是让 agent 逐渐知道哪些提示词、版式和素材组合容易翻车。

---

## 失败记录模板

每次用户指出“哪里需要调整”时，必须记录。记录对象包括失败图片、最终图片、用户反馈、失败原因和修正结果。

每次失败至少记录下面字段：

```text
日期：
产品：
主图目标：
使用版式：
核心卖点：
失败图片：
最终图片：
图片对比：
用户指出的问题：
失败类型：
失败描述：
疑似原因：
修正动作：
下次拦截规则：
最终是否解决：
关联文件：
```

图片路径要尽量使用仓库内或本地可追溯路径。如果图片来自外部生成工具，先记录文件名、版本号或截图说明。

---

## 失败图与最终图对比要求

每条记录必须说明失败图和最终图的差异：

```text
失败图问题：
最终图改动：
保留内容：
删除内容：
新增内容：
下次同类图提前拦截点：
```

对比重点：

- 产品主体是否更真实
- 固定素材是否保持清晰
- 文字是否减少重复
- 价格区是否更清楚
- 背景是否降低 AI 感
- 用户指出的问题是否已经解决

---

## 失败类型

| 类型 | 说明 |
| --- | --- |
| 产品失真 | 外观、比例、颜色、Logo、面板、按钮错误 |
| 证据错误 | 内胆、配件、材质贴片、低糖釜等用错 |
| 参数臆造 | 自动编造价格、功率、材质编号、检测数据 |
| 信息重复 | 标题、贴片、证据区、价格区重复同一卖点 |
| 信息拥挤 | 文字太多，缩略图不可读 |
| 价格弱 | 价格区不醒目，没有成交闭环 |
| AI 感过重 | 塑料感、过度特效、假光影、生成字明显 |
| 平台风险 | 绝对化表达、无法证明承诺、错别字乱码 |

## 常见失败与拦截规则

### 低糖饭证据错误

失败表现：

```text
用普通内胆、饭碗或单个不锈钢盆替代沥糖釜。
```

拦截规则：

```text
只要出现低糖饭/沥糖饭/控糖饭/米汤分离，必须调用常用图片/沥糖釜.jpg，并形成上沥糖釜 + 下配套内胆的证据组合。
```

### 固定素材被 AI 重绘

失败表现：

```text
304/316L 贴片文字变糊，内胆比例变形，颜色变了。
```

拦截规则：

```text
固定证据素材只允许嵌入、裁切、加投影和统一光影，不允许重绘、变形、改字、改颜色。
```

### AI 感过重

失败表现：

```text
背景像梦幻海报，产品像塑料模型，字效过厚，光源不一致。
```

拦截规则：

```text
减少粒子、烟雾、强 3D 字和大面积流光；改用真实摄影棚柔光、轻微环境反射、自然接触阴影和可解释的厨房/台面空间。
```

### 价格区太弱

失败表现：

```text
价格像普通小标签，没有形成下单理由。
```

拦截规则：

```text
转化型主图必须有独立价格区；到手价是价格区最大元素，主价格腰带高度 14%-18%。
```

### 卖点重复

失败表现：

```text
标题写低糖饭，证据贴片又写低糖饭，左侧卖点再写低糖饭。
```

拦截规则：

```text
同一个卖点只出现一次文字表达；证据区用图片证明，不重复文案。
```
