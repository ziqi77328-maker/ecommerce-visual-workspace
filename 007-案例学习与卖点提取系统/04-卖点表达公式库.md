# 卖点表达公式库

用于把优秀案例里的表达方法抽象成可复用公式。

---

## 公式 1：人群 + 痛点 + 产品能力 + 结果

```text
给【人群】，用【产品能力】解决【痛点】，得到【结果】。
```

示例：

```text
给下班晚的人，22分钟快煮好饭，回家不用久等。
```

---

## 公式 2：参数 + 功能 + 用户利益

```text
【参数】+【功能名】，让【用户利益】更明显。
```

示例：

```text
24H预约，早出晚归也能准点开饭。
```

---

## 公式 3：时间窗口 + 权益 + 行动理由

```text
【时间窗口】享【权益】，现在下单更划算。
```

示例：

```text
今晚前享晒图返现，活动期入手更划算。
```

---

## 公式 4：旧问题 + 新方案 + 改善结果

```text
以前【旧问题】，现在【新方案】，实现【改善结果】。
```

示例：

```text
以前做饭要久等，现在快煮模式更省时。
```

---

## 公式 5：核心卖点 + 证据 + 安心感

```text
【核心卖点】，有【证据】支撑，用起来更【安心感】。
```

示例：

```text
母婴级不锈钢内胆，材质看得见，入口更安心。
```

---

## 公式 6：场景画面 + 产品动作 + 情绪收益

```text
在【场景】里，【产品动作】，带来【情绪收益】。
```

示例：

```text
晚饭高峰也不慌，一键快煮，全家准点开饭。
```

---

## 公式 7：主卖点 + 辅助卖点 + 成交权益

```text
【主卖点】解决核心问题，【辅助卖点】补足顾虑，再用【权益】推动成交。
```

示例：

```text
快煮饭省时间，预约保温更省心，限时到手价推动下单。
```

---

## 公式使用规则

- 参数必须来自真实产品信息。
- 价格、补贴、返现、赠品必须来自真实活动信息。
- 不确定时只写占位符，不编造数字。
- 同一个主图最多突出 1 个核心卖点、3-5 个辅助卖点。
- 主卖点讲结果，辅助卖点讲证据，价格权益讲行动理由。
