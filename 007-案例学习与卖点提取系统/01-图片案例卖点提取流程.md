# 图片案例卖点提取流程

用于从一张主图、详情图、竞品图、活动图或优秀案例截图中，提取可复用的卖点和表达结构。

---

## 第一步：客观识别

先记录图片里真实出现的信息，不做主观改写。

```text
案例名称：
图片类型：主图 / 详情图 / 活动页 / 竞品图 / 平台案例
产品类目：
品牌信息：
平台场景：京东 / 天猫 / 抖音 / 小红书 / 视频号 / 其他
活动节点：日常 / 新品 / 大促 / 预售 / 限时 / 清仓
目标人群：
核心价格：
权益信息：
```

---

## 第二步：卖点提取

把图片里的信息拆成“产品功能”和“用户利益”。

| 类型 | 提取问题 | 示例 |
| --- | --- | --- |
| 功能卖点 | 产品有什么能力 | 快煮饭、预约、保温、0涂层 |
| 参数卖点 | 有什么可量化信息 | 3L、316L、99.99%、24H |
| 结果卖点 | 用户得到什么好处 | 更省时、更安心、不串味 |
| 场景卖点 | 在什么场景下有用 | 上班族晚归、全家吃饭、母婴使用 |
| 信任卖点 | 为什么可信 | 品牌背书、平台保障、售后承诺 |
| 成交卖点 | 为什么现在买 | 到手价、返现、赠品、限时补贴 |

---

## 第三步：信息层级判断

判断图片如何安排注意力。

```text
第一眼看到：
第二眼看到：
第三眼看到：
最后推动点击/购买的信息：
```

常见顺序：

```text
品牌信任 -> 核心大卖点 -> 产品主视觉 -> 功能证据 -> 价格权益
```

也可能是：

```text
价格冲击 -> 产品主视觉 -> 核心卖点 -> 赠品权益 -> 售后保障
```

---

## 第四步：卖点表达拆句

把原图里的表达拆成结构，不保留原句。

```text
原表达作用：
表达结构：
情绪方向：
用户利益：
可迁移到哪些产品：
不适合复用的部分：
```

示例：

```text
原表达作用：强调快速完成做饭
表达结构：时间数字 + 功能动作 + 结果利益
情绪方向：省时、轻松
用户利益：下班后不用久等
可迁移到：电饭煲、空气炸锅、微波炉、电压力锅
不适合复用：具体分钟数必须按真实参数填写
```

---

## 第五步：输出可复用结论

每次拆解结束，必须输出这 5 类内容：

```text
1. 本图最核心的卖点：
2. 最值得学习的表达方法：
3. 可迁移公式：
4. 适合当前产品的改写版本：
5. 建议写入仓库的位置：
```
