# 时间表达方法库

用于沉淀大促、预售、限时、补贴、倒计时、返现、赠品等时间相关表达方法。

---

## 时间表达的作用

时间信息不是单纯告诉用户日期，而是给购买动作一个理由。

```text
现在买更划算
错过这次会损失
活动窗口很短
权益有明确截止
新品/首发有阶段感
```

---

## 常见时间表达类型

| 类型 | 表达重点 | 适用场景 |
| --- | --- | --- |
| 限时型 | 只在某个时间段有效 | 秒杀、限时补贴、大促 |
| 倒计时型 | 强调马上结束 | 活动尾声、价格恢复前 |
| 节点型 | 借平台节点制造理由 | 618、双11、年货节、新品日 |
| 预售型 | 先锁权益，后成交 | 高客单新品、爆品预约 |
| 周期型 | 强调每天/每周可持续利益 | 日常使用、长期省时省钱 |
| 错过成本型 | 不买会失去什么 | 返现截止、赠品有限、补贴退坡 |

---

## 可复用句式

### 限时型

```text
限时到手价【价格】
今晚【时间】前享【权益】
活动期下单立减【金额】
本轮补贴限时开放
```

### 倒计时型

```text
距离活动结束仅剩【时间】
错过今晚，价格恢复【原价】
最后【时间】锁定【权益】
```

### 节点型

```text
618 家电焕新价
新品首发限时礼
年货节厨房升级专享
开学季小家电补贴
```

### 预售型

```text
预售先享【权益】
定金锁定【价格/赠品】
提前预约，多享【权益】
```

### 周期型

```text
每天省下【时间】
一周多出【次数/时间】
每顿饭都更省心
```

### 错过成本型

```text
赠品送完即止
补贴结束恢复日常价
本轮返现名额有限
```

## 时间表达检查

使用时间表达前，必须确认：

```text
时间是否真实：
价格是否真实：
权益是否真实：
截止条件是否清楚：
平台规则是否允许：
是否会误导用户：
```

不确定的数据只保留结构，不直接写具体数字。
