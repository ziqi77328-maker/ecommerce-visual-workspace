# 优秀案例学习复用流程

这个流程用于把优秀案例从“看起来不错”拆成“为什么有效”，再转换成自己的主图方案、文案方案和 AI 提示词。

---

## 学习判断框架

每个案例至少从 6 个角度判断。

| 角度 | 判断问题 |
| --- | --- |
| 人群 | 它在对谁说话？ |
| 场景 | 它让用户想到哪个使用画面？ |
| 痛点 | 它放大了什么问题？ |
| 利益 | 它承诺了什么更好结果？ |
| 证据 | 它用什么证明自己可信？ |
| 动作 | 它为什么让用户现在点击或购买？ |

---

## 常见有效策略

### 痛点放大型

先让用户意识到问题，再给出产品解决方案。

```text
适合：老产品换新、功能升级、解决明确麻烦的产品
结构：痛点场景 + 产品能力 + 结果改善
```

### 结果承诺型

直接展示用户最想要的结果。

```text
适合：快煮、低糖、保温、清洁、容量、母婴安全
结构：目标结果 + 关键功能 + 可信证据
```

### 稀缺时间型

把活动时间、库存、补贴、返现做成行动理由。

```text
适合：大促、预售、秒杀、清仓、新品首发
结构：时间窗口 + 成交权益 + 错过成本
```

### 身份代入型

让用户觉得这张图说的是自己。

```text
适合：上班族、小家庭、母婴家庭、租房党、长辈使用
结构：人群身份 + 日常麻烦 + 产品解决
```

### 对比反差型

用前后变化、旧新方案、价格对比制造冲击。

```text
适合：升级款、替代方案、强促销、功能差异明显的产品
结构：旧状态 + 新方案 + 明显差别
```

---

## 复用步骤

1. 只保留案例的表达结构，不保留原品牌和完整原文。
2. 把原案例参数替换成当前产品真实参数。
3. 把原案例人群换成当前产品最适合的人群。
4. 把原案例活动表达换成当前平台真实活动。
5. 把原案例视觉结构改成当前产品适合的主图结构。
6. 输出 3 个版本：保守版、转化版、强促销版。
