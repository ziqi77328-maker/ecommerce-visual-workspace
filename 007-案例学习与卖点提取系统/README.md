# 案例学习与卖点提取系统

这个目录用于把外部优秀主图、详情图、活动页截图、竞品图和平台案例，拆成可复用的卖点表达、时间表达、视觉结构和 AI 生成提示词。

核心目标不是照抄案例，而是提取它为什么有效，再转换成适合当前产品、平台和活动节点的表达方法。

---

## 使用顺序

1. 先用 [01-图片案例卖点提取流程.md](01-图片案例卖点提取流程.md) 做客观拆解。
2. 再用 [02-优秀案例学习复用流程.md](02-优秀案例学习复用流程.md) 判断它为什么有效。
3. 学习主图视觉时，用 [07-视觉表达学习流程.md](07-视觉表达学习流程.md) 拆主体、特效字、卖点卡、背景和色彩。
4. 对信息密度高的主图，用 [06-主图证据链与视觉层级检查表.md](06-主图证据链与视觉层级检查表.md) 检查卖点是否有视觉证据。
5. 遇到大促、限时、预售、补贴、倒计时时，补充到 [03-时间表达方法库.md](03-时间表达方法库.md)。
6. 把可复用的表达公式沉淀到 [04-卖点表达公式库.md](04-卖点表达公式库.md)。
7. 每次完整学习一个案例时，用 [05-案例拆解记录模板.md](05-案例拆解记录模板.md) 记录。
8. 最后把能直接生成主图的内容，按规则同步到 `001-项目案例库`、`003-产品结构排版库`、`004-卖点表达库`、`005-价格与权益表达库`、`006-AI关键词库`。

---

## 每次发图给 AI 时的标准指令

```text
请按仓库里的 007-案例学习与卖点提取系统，学习这张图。

输出：
1. 客观信息提取
2. 核心卖点拆解
3. 时间表达/价格权益表达
4. 视觉表达学习：主体、特效字、卖点卡、背景、色彩、信息动线
5. 证据链与视觉层级
6. 为什么有效
7. 可复用表达公式和视觉方法
8. 适合我当前产品的改写版本
9. 应写入仓库哪些文件
```

如果要直接写入仓库，补一句：

```text
请把本次学习结果写入对应知识库，避免重复沉淀。
```

---

## 输出分层

| 层级 | 目的 | 写入位置 |
| --- | --- | --- |
| 案例记录 | 保留完整拆解过程和判断 | `007-案例学习与卖点提取系统/案例学习记录.md` 或 `001-项目案例库` |
| 卖点说法 | 沉淀可迁移的文案表达 | `004-卖点表达库` |
| 时间权益 | 沉淀限时、预售、补贴、返现、赠品表达 | `005-价格与权益表达库` 或本目录时间库 |
| 视觉结构 | 沉淀版式、区域、信息优先级 | `003-产品结构排版库` |
| 视觉表达 | 沉淀主体光感、特效字、背景、色彩、卖点卡的学习方法 | `007-案例学习与卖点提取系统` |
| 证据链 | 沉淀主卖点如何被局部图、参数、过程图证明 | `007-案例学习与卖点提取系统` 或 `003-产品结构排版库` |
| AI提示词 | 沉淀可直接复制生成的完整提示词 | `006-AI关键词库` |

## 反照抄原则

- 不复制原案例的完整文案。
- 不照搬竞品品牌名、价格、活动承诺和专属权益。
- 只提取结构、节奏、利益表达、时间压力、证据顺序和视觉组织方式。
- 改写时必须换成自己的产品参数、平台规则、活动信息和目标人群。
- 如果原案例的表达依赖无法验证的数据，不直接复用，只保留表达结构。
